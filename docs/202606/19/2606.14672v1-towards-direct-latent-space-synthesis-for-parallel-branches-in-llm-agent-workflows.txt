# Towards Direct Latent-Space Synthesis for Parallel Branches in LLM-Agent Workflows 

Shikun Liu 1, Mufei Li 1, Dongqi Fu 2, Haoyu Wang 1, Yinglong Xia 2, Hong Li 2, Hong Yan 2, Pan Li 11Georgia Institute of Technology, 2Meta Large language models increasingly serve as execution engines for agentic systems, yet they still consume context through a sequential text interface. This creates a mismatch with modern structured agent workflows, in which independent branches explore subtasks, retrieve evidence, or generate candidate solutions before a final synthesis step. Existing systems typically merge these branches by concatenating their textual outputs, which discards the parallel structure and incurs redundant prefill computation. In this work, we introduce Parallel-Synthesis, a plug-and-play framework that enables a synthesizer to directly consume the KV caches produced by parallel worker agents. Parallel-Synthesis combines a cache mapper that calibrates independently generated branch caches with a fine-tuned synthesizer adapter that enables generation from this non-sequential cache interface. We train Parallel-Synthesis using data that exposes the synthesizer to parallel cache contexts, teaches aggregation across cached branches, and distills reasoning behavior from standard text-concatenation-based synthesis. Across nine downstream datasets spanning math, science QA, code generation, GAIA, and multi-agent database diagnosis, Parallel-Synthesis matches or outperforms text-based synthesis on seven datasets and remains close on the other two. It also reduces time-to-first-token by 2.5 ×–11 ×, suggesting that direct cache-based synthesis is a promising interface for more native and efficient synthesis over parallel agent branches.

# 1 Introduction 

Large language models (LLMs) serve as the core execution engines for agentic systems due to their advanced reasoning and tool-use capabilities (Yao et al., 2022; Wang et al., 2024b; Schick et al., 2023). Despite these advances, LLMs still operate through a sequential context interface: the entire agent state, including message history, tool outputs, retrieved evidence, and intermediate reasoning, is serialized into a single ordered token prefix for next-step execution. While this interface has been highly effective for scaled-up autoregressive training, it provides only an indirect way to represent the rich dependency structure that arises in agentic computation. Moreover, as histories grow, such serialization also imposes artificial dependencies among otherwise unrelated information, lengthening the prefix that needs to be processed and making it harder for the model to focus on the evidence most relevant to the current query (Liu et al., 2024; Shi et al., 2023; Zhang et al., 2024). This mismatch becomes especially clear when agent workflows contain parallel structure. Many complex tasks are better represented as directed acyclic graphs (DAGs), where independent subtasks, retrieval branches, candidate solutions, or tool calls can be executed separately and merged later. Such parallel-then-synthesize patterns have emerged as an effective orchestration strategy for improving both reliability and efficiency in complex problem solving. Even in single-agent settings, mathematical reasoning and code generation systems often improve reliability by sampling multiple test-time trajectories and then selecting among candidate solutions or aggregating sampled reasoning chains (Chen et al., 2021; Brown et al., 2024; Wang et al., 2023). In multi-agent systems, this structure becomes more explicit: parallel sub-agents can gather evidence from different perspectives or solve independent subtasks before a downstream agent merges their findings (Anthropic, 2025; Chai et al., 2025; Li et al., 2025; OpenAI, 2026; Anthropic, 2026). However, because LLMs natively consume only linearized contexts, synthesizing multiple independent branches 1

typically requires concatenating their outputs into a single expanded text prompt for the synthesizer. This text-based communication interface discards the original DAG structure of the computation and incurs redundant prefill cost by re-encoding information that has already been processed during worker decoding. Since such non-sequential agent workflows become increasingly common, it is timely to move beyond treating context as a single sequential prefix and to consider interfaces that preserve independence among branches while supporting efficient and accurate synthesis. One natural engineering mitigation is to summarize each branch and pass the summaries to the synthe-sizer (Anthropic, 2026; Sun et al., 2025; Kang et al., 2025). While this can reduce the long-context burden, it still relies on natural-language communication: summaries must be generated, serialized, and re-encoded through prefill, and the summarization step may discard details that are important for downstream judgment. In this work, we study a more direct alternative, one that points toward a more fundamental agent capability: can a synthesizer agent directly consume the latent states of parallel worker branches, rather than relying on concatenated plain-text outputs? Such an interface could reduce redundant computation while better preserving the parallel structure of non-sequential agent workflows. Communicating through KV caches instead of explicit text is related to recent work on latent-space com-munication between agents. Most existing formulations focus on one-to-one latent transfer, where a latent state or cache produced under one context is passed to another agent (Zou et al., 2025; Ye et al., 2025; Fu et al., 2025). Our setting instead studies many-to-one synthesis over parallel agent branches, where the downstream synthesizer must jointly consume and integrate several independently produced branch states. Another adjacent line of work studies parallel encoding and cache reuse for long-context RAG, where retrieved document chunks are independently encoded into KV caches and later reused during generation (Yang et al., 2025b,d; Ratner et al., 2023; Yao et al., 2025). While related in mechanism, this setting differs from ours in how cached contexts are constructed, how KV states are obtained, and what the downstream model is required to do. We defer a more detailed discussion to Sec. 2.2. In this work, we present Parallel-Synthesis , a plug-and-play synthesizer framework for direct synthesis over the cached states of parallel agent branches. Unlike standard text-based synthesis, which concatenates branch outputs and prefills their raw text again, Parallel-Synthesis enables the downstream synthesizer to reuse caches produced by parallel worker agents for subsequent generation. Since this departs from the standard serialization assumed by standard LLM pipelines, Parallel-Synthesis employs a cache mapper to calibrate independently produced branch caches, together with a synthesizer LoRA that adapts the model to this non-sequential cache interface. To support this adaptation, we construct training datasets with complementary goals: 1) large-scale continued-pretraining-style dialogue data for broad adaptation to parallel cache contexts; 2) multi-source synthesis tasks that teach the model to aggregate information from multiple cached contexts when responding to queries; and 3) distillation data from a standard text-concatenation-based synthesis pipeline to strengthen reasoning and judgment over complex agent trajectories. We evaluate Parallel-Synthesis on downstream tasks spanning standard problem-solving and agentic branch-and-synthesize settings, including mathematical reasoning, scientific QA, code generation, tool-use agentic QA, and multi-agent diagnostic reasoning. Across these diverse settings, Parallel-Synthesis largely preserves or improves the quality of standard text-based synthesis while substantially improving efficiency: it matches or outperforms text-based synthesis on 7 of 9 datasets, with only small gaps on the remaining two, and reduces time-to-first-token (TTFT) by 2.5 ×–11 × by avoiding the re-prefill of branched worker outputs. Notably, Parallel-Synthesis achieves gains on several reasoning-heavy tasks, suggesting that direct latent-state synthesis can be not only faster but also competitive, and sometimes beneficial, for integrating parallel reasoning trajectories.

2.2 Difference from RAG-Style Parallel Encoding 

Several prior works study KV-cache reuse for long-context RAG, where retrieved documents are independently encoded into cache blocks and later reused for query-conditioned generation (Ratner et al., 2023). To align with our notation, let each retrieved document chunk be denoted by zj . These methods encode each zj into a cache block and later reuse the collection of cached chunks during downstream decoding. Since the cached chunks are not produced as a single contiguous prefix, such methods often introduce auxiliary mechanisms, such as partial recomputation (Yao et al., 2025), attention calibration (Yang et al., 2025d), or linking tokens (Yang et al., 2025b), to make independently encoded caches jointly usable. Our setting is related but differs in several important ways. First, in RAG-style cache reuse, zj typically denotes an isolated document chunk encoded without a worker-specific prefix context cj or prior workflow state. In parallel agent synthesis, by contrast, zj denotes the output of a worker branch generated under its own context cj , which may include a local instruction, reasoning trajectory, tool interaction, or intermediate observation. Second, RAG chunks zj are often obtained by segmenting a longer document (Ratner et al., 2023; Ma et al., 2025), and therefore may not be semantically self-contained. This can leave unresolved dependencies across chunks, requiring mechanisms that repair or calibrate attention across cached blocks. In our setting, zj

is usually a more coherent unit of agent progress, such as a candidate solution, branch summary, or subtask result. This makes branch caches more naturally suited to parallel processing and downstream synthesis than arbitrary document chunks. The downstream objective is also different. RAG-style cache reuse mainly aims to help the model retrieve and use evidence from cached chunks {zj }mj=1 for a final query. Parallel agent synthesis instead requires the synthesizer to compare, judge, reason over, aggregate, and summarize heterogeneous worker outputs {zj }mj=1 

under a broader range of instructions. Thus, while RAG-style cache reuse provides useful technical inspiration, it targets a different communication pattern and downstream objective from the parallel agent branching synthesis setting studied in this work. Consistent with this distinction, our experiments in Sec. 4 show that directly adapting RAG-style cache-reuse methods to parallel agent synthesis leads to weak downstream performance.

3.2 Model Training 

A parallel cache-based synthesizer must acquire two complementary abilities. First, it should learn to read 

and interpret parallel worker caches. Second, it should preserve its reasoning ability when using those cached trajectories under the synthesizer instruction. We therefore train two complementary post-training tracks 5from the same base model and merge them afterward: Track 1 provides broad and scaled-up adaptation to parallel cache contexts, while Track 2 targets realistic synthesis and reasoning over parallel agent trajectories through distillation from the text-serialization route.

> Track 1: General adaptation over parallel contexts.

This training track equips the synthesizer to process indepen-dently encoded parallel contexts rather than a traditional serialized sequence. We use two complementary data types: continued-pretraining-style data and supervised fine-tuning data on explicit parallel-context tasks. The former teaches the model naturally resume generation from multi-turn histories structured as parallel KV cache. The latter provides explicit, task-oriented supervision, compelling the model to actively synthesize information across these parallel branches to resolve queries. The dataset statistics are included in Appendix A.1.1. 

1) Continued-pretraining-style adaptation. We use large-scale multi-turn dialogue data, WildChat (Zhao et al., 2024b), UltraChat (Ding et al., 2023), and LMSYS-Chat (Zheng et al., 2023), to adapt the model to generate from non-sequential prefix caches. We treat the final user query as u and the final assistant response as y⋆.The preceding dialogue turns are encoded independently as parallel caches rather than concatenated into one serialized history. Since the target response may depend only weakly on earlier turns, this stage primarily acts as an architectural warm-up that adapts the model to conditioning on parallel contexts. 

2) Supervised fine-tuning on parallel-context tasks. We construct training samples from tasks whose inputs can be naturally decomposed into independent, self-contained semantic blocks, and whose solutions explicitly require the model to follow the instruction and read and understand the provided parallel contexts. For agentic tool-use data such as Toucan (Xu et al., 2025) and DTA-Tool (Zhu et al., 2025a), we extract parallel tool-use trajectories. Here, u is the user query, each tool-call–observation pair is treated as a parallel branch output zj , the preceding agent history defines its prefix context cj , and y⋆ is the next agent response that interprets and uses these tool results. For in-context learning data like FLAN (Wei et al., 2022), u is the target question, y⋆ is the answer, and preceding few-shot exemplars serve as the parallel contexts. Finally, for multi-document QA such as 2Wiki (Ho et al., 2020), u is the query and individual source documents form the parallel branches. We construct y⋆ as a reasoning trace that first summarizes the provided contexts, analyzes their relevance to u, and then deduces the target answer.  

> Track 2: Distillation from Text-Based Synthesis Trajectories.

This track targets the richer synthesis behavior required in realistic agentic settings, where the synthesizer must reason over, compare, and judge prior sub-agent progress under detailed task instructions. Although Track 1 teaches the model to read and use parallel caches, it does not provide enough supervision for higher-level reasoning over the evidence encoded in them, which limits end-task synthesis performance in practice. We therefore distill synthesis trajectories from the text-serialization route, using them as teacher signals for reasoning-intensive synthesis. Concretely, we use BrowseComp (Wei et al., 2025) to construct parallel worker trajectories and synthesis supervision. For each sample, we first generate three parallel worker rollouts and then run the synthesizer under the text-based synthesis route, where the worker outputs are serialized as text context. Although these synthesis traces are constructed from three worker trajectories, the trained synthesizer is not tied to this rollout count and generalizes to different numbers of parallel trajectories at evaluation time. We then collect the resulting synthesis trajectories and filter out failed or low-quality cases, retaining only those with reliable reasoning and valid final responses as supervision targets y⋆ for cache-based training. Each retained example is converted into our cache-based training format: for worker j, cj is the worker-local history before its final output turn, including prior messages, tool calls, tool observations, and intermediate reasoning, and zj is the final output communicated to the synthesizer. # 4 Experiments  

> Evaluation datasets

We evaluate Parallel-Synthesis on nine datasets across four domains: (1) Mathematical reasoning: AIME 2024/2025 (Maxwell-Jia, 2024; Zhang and Math-AI, 2025) and GSM8K (Cobbe et al., 2021); (2) Scientific QA: GPQA (Rein et al., 2024) and MedQA (Yang et al., 2025a); (3) Code generation: HumanEval-Plus and MBPP-Plus (Liu et al., 2023); and (4) Multi-turn agentic benchmarks with tool use: GAIA (Mialon et al., 2023) and the database task from MARBLE (Zhu et al., 2025b). In all tasks except MARBLE Database, parallel worker agents are used to explore multiple candidate solution or search trajectories. In MARBLE Database, worker agents instead handle decomposed subtasks to independently analyze possible root causes of database errors. We use 5 parallel worker agents for MARBLE Database task since it is determined by fixed sub-tasks and 3 parallel workers for the rest datasets. These benchmarks cover diverse settings where parallel-then-synthesize execution is useful. Importantly, none of them overlap with the post-training datasets, so all results reflect out-of-domain generalization. More details are provided in Appendix A.1.2.

RQ2: Does Parallel-Synthesis perform genuine reasoning-based synthesis over parallel agent trajectories, beyond simple answer aggregation? 

An important question is whether Parallel-Synthesis uses parallel agent trajectories as rich synthesis context, rather than treating them as a set of final answers to be aggregated by voting. The Voting baseline provides a direct test for this distinction. Across 8 of the 9 datasets, Parallel-Synthesis outperforms pure majority voting, indicating that its gains cannot be explained by simply recovering the most common worker answer. MedQA is the only exception, as this multiple-choice task is particularly amenable to majority voting. These results suggest that Parallel-Synthesis can exploit trajectory-level signals beyond workers’ final answers, including intermediate reasoning steps, evidence quality, and disagreements among workers. This capability is especially important for tasks such as GAIA, where the correct answer is not always supported by the majority of workers’ final answers. Case studies in Appendix A.4 show that Parallel-Synthesis can ground its synthesis in intermediate evidence from parallel agents; in some examples, it even performs an additional tool call for verification when needed. By contrast, although KVLINK performs reasonably on some datasets, it lags behind Parallel-Synthesis overall and often jumps directly to a final answer without reasoning over the worker trajectories or provided instruction. This suggests that post-training for RAG-style cache reuse may not be sufficient for the trajectory-level reasoning and instruction-following ability needed for parallel-agent synthesis. 

RQ3: Importance of post-training for robust and general parallel cache synthesis. 

Post-training is critical for making direct cache reuse reliable in parallel synthesis. Although Parallel-Synthesis is not trained on any of the nine downstream benchmarks, it transfers effectively across all of them, suggesting that the learned synthesis behavior generalizes beyond the training distribution. In contrast, inference-time cache-reuse methods such as APE, CacheBlend, and the no-training variant of Parallel-Synthesis perform substantially worse, whereas the post-trained KVLINK baseline is comparatively stronger. 4.2 Ablation Studies  

> Ablation on different post-training checkpoints.

We compare four variants on one representative dataset from each task category: no post-training, Track 1 only, Track 2 only, and sequential tuning where Track 2 is applied after Track 1. As shown in Table 3, Track 1 tends to perform better on simpler single-turn problem-solving tasks, such as HumanEvalPlus and GPQA, whereas Track 2 is stronger on reasoning-heavy math and more complex agentic tasks. This pattern is consistent with the data composition and intended capabilities of the two training tracks described in Sec. 3.2. The final merged checkpoint achieves the best overall balance, while sequential tuning typically underperforms both single-track checkpoints and the merged model. This suggests that sequential tuning fails to preserve the complementary strengths learned from the two tracks.  

> Ablation on framework modules.

We further examine the contributions of the learnable cache mapper and synthesizer LoRA. Since the no-training variant and inference-time cache-reuse baseline APE perform poorly, synthesizer adaptation is necessary for parallel cache synthesis. Table 3 further shows that adding the learnable cache mapper consistently improves over the synthesizer-LoRA-only variant, indicating that cache mapping provides complementary benefits beyond LoRA adaptation.  

> Ablation on communicated trajectory information.

In single-turn tasks, each worker produces a single response, so the communicated context is unambiguous. In multi-turn ReAct-style tasks, however, a worker trajectory contains instructions, intermediate model responses, tool calls, and tool observations, raising the question of which parts should be passed to the synthesizer. Table 4 compares three choices on GAIA Level 1: final output , which passes only the worker’s final response; each-turn output , which passes all model-generated responses but excludes instructions and tool observations; and full trajectory , which passes the complete worker history. Full trajectories achieve the highest accuracy by preserving tool observations, but incur the largest latency, especially for Text-Serialization. Final outputs provide the best accuracy-efficiency tradeoff and are used as our default. Each-turn outputs perform worse, likely because removing tool observations leaves fragmented, weakly grounded traces. These results show that Parallel-Synthesis can synthesize from different levels of trajectory information while benefiting most from concise, coherent worker outputs. 

# 5 More Related Works  

> LLM-based Multi-agent Systems (MAS).

Recent LLM-based multi-agent systems increasingly move beyond sequential execution (Yao et al., 2022; Zhang et al., 2024) by organizing agent computation as structured workflows, often in the form of computational graphs or DAGs, where LLM/tool nodes correspond to intermediate subtasks and edges specify execution dependencies (Zhuge et al., 2024; Zhang et al., 2025b; Yu et al., 2025; Qin et al., 2025). Within these graph-structured workflows, the parallel-then-synthesize pattern appears in several forms. First, some workflows parallelize solution and rationale generation, where multiple 9agents or model calls independently solve the same problem and their candidate answers are later aggregated through voting, debate, ranking, or response fusion (Du et al., 2024; Li et al., 2024; Jiang et al., 2023; Wang et al., 2024a). Second, some workflows parallelize tool use and open-ended exploration, where independent retrieval/API calls or tool-using subagents are executed along different search directions before a lead agent synthesizes their observations (Kim et al., 2024; Anthropic, 2025; Wu et al., 2025; Miloradović et al., 2023; Zhang et al., 2025c; Chai et al., 2025). Lastly, some works parallelize over input contexts: long documents or retrieved evidence are split into chunks, processed by separate agents and then aggregated into a final answer (Zhao et al., 2024a; Zhou et al., 2025; Liu et al., 2025).  

> Latent communication in multi-agent systems.

Many work studies how LLM agents can communicate beyond decoded texts. They mainly fall into two lines: one uses latent representations as compact thought-level messages for direct agent communication (Zheng et al., 2025; Zou et al., 2025; Du et al., 2025), while the other treats KV caches as transferable execution states, enabling agents or models to reuse, share, or fuse internal context without decoding it into text (Ye et al., 2025; Shi et al., 2025; Fu et al., 2025). These works mainly study pairwise or sequential inter-agent transfer, where one agent’s internal state is passed to another agent. In contrast, our setting targets the many-to-one convergence pattern in non-sequential workflows: several independently generated branch caches must be jointly interpreted by a downstream synthesizer. A recent work (Jin et al., 2026) investigates reusable latent primitives for multi-agent collaboration, including parallel branches voting and selection, whereas we focus on training a general parallel-agent synthesizer. 

# 6 Conclusion and Future Direction 

We introduced Parallel-Synthesis, a plug-and-play framework that enables a downstream synthesizer to directly consume the KV caches produced by parallel worker agents. Through post-training, Parallel-Synthesis performs effectively across a diverse set of parallel-then-synthesize tasks, matching or surpassing text-based synthesis while substantially reducing redundant computation. Parallel-Synthesis is not yet ready to replace text-based communication in all agent frameworks, especially for highly complex structured workflows. Nevertheless, our results provide encouraging evidence that direct cache-based synthesis is a promising interface for parallel agent branches. If this direction succeeds, DAG-structured workflows could be executed and merged more natively, without the repeated computational overhead of text serialization and context refilling. Achieving this goal will likely require scaling post-training to broader and larger collections of agent workflow data, covering more diverse tool-use patterns, branching structures, and synthesis objectives.

• LMSYS-Chat (Zheng et al., 2023): A real-world LLM conversation dataset collected from the Vicuna demo and Chatbot Arena. It contains conversations with multiple state-of-the-art LLMs and is useful for modeling natural user–assistant interactions. 

• UltraChat (Ding et al., 2023): A large-scale synthetic multi-turn instructional dialogue dataset generated with ChatGPT-style user and assistant roles. It covers broad open-domain instruction-following scenarios and is designed for chat model training. 

• Toucan-single-turn (Xu et al., 2025): A subset of Toucan-1.5M, a large tool-agentic dataset synthesized from real-world MCP environments. Here, we filter out only single-turn interaction setting but with parallel toolcalls. We train the model to generate corresponding response given the parallel tool-calling execution. 

• Toucan-multi-turn (Xu et al., 2025): The multi-turn subset of Toucan, containing tool-agent trajectories extended to multi-turn interactions. It is useful for training models to plan tool use conditioned on previous user–agent interactions. 

• DTA-Tool (Zhu et al., 2025a): A parallel tool invocation dataset built for DTA-Llama. It converts traditional tool-search trajectories into DAG-style parallel tool invocation traces, where the model learns to divide a task into parallel tool-using subtasks and aggregate the results. 

• FLAN (Wei et al., 2022): An instruction-tuning collection that formats many NLP tasks with natural language instructions, including zero-shot, few-shot, and chain-of-thought templates. In your setting, it is suitable for constructing current-question/current-answer pairs with previous in-context examples. 

• 2WikiMultiHopQA (Ho et al., 2020): A multi-hop question answering dataset built from structured Wikidata and unstructured Wikipedia evidence. It provides explicit evidence and reasoning paths, making it useful for training or evaluating context-grounded multi-hop reasoning.

> Code Generation

• HumanEval-Plus (Liu et al., 2023): It is a Python code-generation benchmark for evaluating the functional correctness of LLM-synthesized programs via execution against much more exhaustive unit tests than typical coding datasets. It pairs HumanEval-style programming prompts (function signature + docstring) with automatically generated, type-aware, adversarially diverse test inputs to better catch subtle logical errors that slip past sparse test suites, making it a stricter measure of whether models produce semantically correct, runnable code rather than merely plausible-looking solutions. 

• MBPP-Plus (Austin et al., 2021; Liu et al., 2023): It extends the Mostly Basic Python Problems benchmark, which contains short Python programming tasks designed to be solvable by entry-level programmers. Compared with the original MBPP tasks, which include only a small number of tests, MBPP-Plus augments the benchmark with substantially richer unit tests, making it a more rigorous evaluation of functional correctness, edge-case handling, and the robustness of generated Python code. 

> Scientific QA

• GPQA-Diamond (Rein et al., 2024): It is the hardest subset of the GPQA benchmark, consisting of 198 graduate-level multiple-choice questions written by domain experts in physics, biology, and chemistry. It is designed to be “Google-proof,” with carefully engineered distractors that require deep conceptual understanding, cross-topic reasoning, and multi-step scientific inference rather than surface pattern matching. 

• MedQA (Yang et al., 2025a): It comprises real medical licensing–style multiple-choice questions that evaluate biomedical knowledge and clinically grounded reasoning. Solving items often requires integrating the problem narrative with domain expertise to make diagnostic or management decisions under plausible distractors, making it a practical proxy for professional-level medical QA performance.  

> GAIA

(Mialon et al., 2023): A benchmark for general AI assistants that tests whether a system can solve real-world, conceptually simple (for humans) questions that nonetheless require multi-step reasoning, web browsing, multimodal understanding, and tool use (e.g., reading files, searching, computing). In our setting, we use 5 max toolcalls for each parallel worker.  

> MARBLE Database

(Zhu et al., 2025b): It is a multi-agent database diagnosis task from MultiAgentBench/-MARBLE, where a PostgreSQL database exhibits performance anomalies and agents must collaboratively identify the underlying root cause. Solving the task requires agents to inspect database system views such as 16 query statistics and lock information, communicate across multiple rounds, and synthesize evidence across possible causes including large data fetching or insertion, lock contention, redundant indexes, and vacuum-related performance issues. In our setting, we mainly let each parallel worker agent investigate one root cause and pass its analysis to the synthesizer and let the synthesizer to decide the final root causes. 

A.2 Experimental Configuration  

> Post-Training.

During post-training, for both tracks, we jointly optimize the cache affine mapper and a judger LoRA adapter. The LoRA adapter uses rank 16, alpha 32, and dropout 0.0, and is applied to the attention projection modules q_proj , k_proj , v_proj , and o_proj . We optimize the trainable modules with AdamW using a learning rate of 1 × 10 −4 and weight decay 0.0. We use 5 A100 for training with GPU memory 80G. Post-training for track 1 takes several days and for track 2 takes hours less than a day.  

> Evaluation.

Note that for the GAIA dataset, we use the open validation set as our downstream performance evaluation. This is because there is no visible answer for the test set. And using GAIA validation for evaluation is a common practice in academic papers (Nguyen et al., 2025; Soni et al., 2026; Zhang et al., 2025a). For all datasets inference, we set the max new tokens for the parallel worker and the synthesizer agent to 4096 by default except the AIME datasets we set to 6000. We use 5 max tool steps for GAIA and MARBLE dataset. Regarding baselines, for the APE baseline, we use the default scale and temperature in their code as both 0.9. For the CacheBlend, we use 0.15 recomputation rate.

A.4 Cases Studies of Parallel-Synthesis 

Case study: synthesis without majority agreement. The following AIME 2024 example illustrates that Parallel-Synthesis can synthesize useful partial reasoning from multiple workers even when none of the workers produces the correct final answer. 

AIME 2024 Case Study: Correct Synthesis without a Majority Answer  

> Problem.

Find the number of triples of nonnegative integers (a, b, c ) satisfying a + b + c = 300 and 

a2b + a2c + b2a + b2c + c2a + c2b = 6 ,000 ,000 . 

> Gold answer:

601 Synthesizer prediction: 601  

> No-majority worker outputs.

The three parallel workers produce three distinct final answers, so there is no majority answer to vote for.                               

> Source Final answer Useful evidence in the trace
> Sub-agent 1 2Correctly reduces the original equation to 100( ab +bc +ca )−abc = 2 ,000 ,000 ,but does not complete the count. Sub-agent 2 100 Finds the decisive factorization (a−100)( b−100)( c−100) = 0 , but does not produce the correct final count. Sub-agent 3 200 Checks valid examples such as (100 ,100 ,100) and (200 ,100 ,0) , providing sanity checks for the reduced equation.

Complementary evidence extracted from workers  

> Sub-agent 1: equation reduction.

Using a + b + c = 300 , the worker derives 

300( ab + bc + ca ) − 3abc = 6 ,000 ,000 ,

and hence 

100( ab + bc + ca ) − abc = 2 ,000 ,000 . 

> Sub-agent 2: key factorization.

The worker observes that 

(a − 100)( b − 100)( c − 100) = abc − 100( ab + bc + ca ) + 100 2(a + b + c) − 100 3.

Since a + b + c = 300 and 100( ab + bc + ca ) − abc = 2 ,000 ,000 , this becomes 

(a − 100)( b − 100)( c − 100) = 0 . 

> Sub-agent 3: sanity checks.

The worker verifies examples such as (200 , 100 , 0) and (100 , 100 , 100) ,both of which satisfy the reduced equation 100( ab + bc + ca ) − abc = 2 ,000 ,000 .

22 Synthesizer reconstruction 

The synthesizer combines the partial evidence rather than voting over final answers. It first identifies the critical structural evidence from Sub-agent 2, as shown by its opening reasoning:             

> “From my earlier analysis, I found that the equation simplifies to (a−100)( b−100)( c−100) = 0 .This means that at least one of the variables a,b, or cmust be equal to 100 .”

Although Sub-agent 2 does not produce the correct final count, this factorization correctly characterizes the solution set. The synthesizer then uses this evidence to complete the counting argument. Let 

A = {a = 100 }, B = {b = 100 }, and C = {c = 100 }. If a = 100 , then b + c = 200 , giving 201 

nonnegative solutions. Therefore |A| = |B| = |C| = 201 . For any pair, e.g., a = b = 100 , the remaining variable is forced to be 100 , so 

|A ∩ B| = |A ∩ C| = |B ∩ C| = 1 , |A ∩ B ∩ C| = 1 .

By inclusion-exclusion, 

|A ∪ B ∪ C| = 3 · 201 − 3 · 1 + 1 = 601 .

Thus, by using the correct intermediate evidence from a worker with an incorrect final answer, the synthesizer obtains the correct final answer: 

601 . 

> Takeaway.

This example shows that Parallel-Synthesis can solve cases where parallel agents do not agree on a final answer and none of the worker final answers is correct. The synthesizer succeeds by integrating complementary latent reasoning signals: one worker provides the reduced symmetric equation, another provides the key factorization, and a third supplies useful validity checks.
