## 摘要
符号回归旨在从数据中揭示显式的科学定律。最近的方法利用大语言模型（LLM）从背景文本引导变异，这比随机遗传编程更具方向性。然而，精确的符号恢复需要语义引导和显式结构，以便通过有效的符号表示进行领域信息搜索。当前基于LLM的系统仍然是结构盲的：它们在黑箱候选项中进行选择，缺乏局部变异的显式机制，并依赖脆弱的系数拟合，这可能导致低估正确的骨架。我们提出FunctionEvolve，一个使用表达式树组织整个搜索过程的进化框架：结构摘要促进多样化的父代选择，局部树编辑保留有用的子表达式，结构感知拟合通过分解、约束和简化系数实现更可靠的评分。它仅使用基本函数族，没有限制泛化能力的额外领域特定规则。在LLM-SRBench的129任务合成子集上，使用Claude Opus 4.6的FunctionEvolve恢复了107个精确形式，达到82.9%的SA@50，是同等骨干基线的4.5倍，以及55.8%的SA@1，是此前最强已发表top-1结果的3.6倍。消融实验表明，结构可见搜索是可靠恢复的核心，而LLM引导的优化和结构感知的系数优化作为关键的提议和评分机制。我们还对该基准进行了审计，并表明其材料科学子集中的共线性导致了可辨识性问题。

## Abstract
Symbolic regression aims to uncover explicit scientific laws from data. Recent methods use LLMs to guide mutation from background text, which is more directed than random genetic programming. However, exact symbolic recovery requires both semantic guidance and explicit structure, so that domain-informed search are carried out through valid symbolic representation. Current LLM-driven systems remain structure-blind: they select among opaque candidates, lack explicit mechanisms for local mutation, and rely on brittle coefficient fitting that can undervalue correct skeletons. We propose FunctionEvolve, an evolutionary framework using expression trees to organize the whole search: structural summaries promote diverse parent selection, local tree edits preserve useful subexpressions, and structure-aware fitting decomposes, constrains, and simplifies coefficients for more reliable scoring. It uses only elementary function families, without additional domain-specific rules limiting generalization. On the 129-task synthetic subset of LLM-SRBench, FunctionEvolve with \emph{Claude Opus 4.6} recovers 107 exact forms, reaching 82.9% SA@50, 4.5x above same-backbone baselines, and 55.8% SA@1, 3.6x above the strongest previously published top-1 result. Ablations show that structure-visible search is central to reliable recovery, with LLM-guided refinements and structure-aware coefficient optimization serving as essential proposal and scoring mechanisms. We also audit the benchmark and show that collinearity in its materials-science subset creates identifiability issues.

## 2. 方法论
#### 核心思想
- 将SR问题形式化为：给定数据集 \(D = \{(x_i, y_i)\}\) 和任务上下文 \(T\)，搜索最匹配的表达式 \(\tilde{f}\)。
- 使用**表达式树（AST）** 作为统一的显式结构表示，贯穿**选择（Selector）**、**变异（Mutator）** 和**系数优化（Optimizer）** 三个核心环节。

#### 关键技术细节
1. **初始化阶段（Generator）**  
   - 根据任务上下文和LLM知识生成领域知识（公式先验和结构启发），并初始化一组种子表达式（含通用初等函数结构和知识引导结构）。
2. **迭代进化阶段**
   - **Selector（选择器）**：基于进化树摘要（AST结构、参数数、深度、算子数、训练NMSE、拟合参数值）和历史选择记录，由LLM选择5个结构化多样、简洁且低误差的父代表达式。避免纯贪婪，防止重复扩展相似分支。
   - **Mutator（变异器）**：对每个选中的父代，实施三类操作：
     - **规则添加**：通过预设的初等函数库（线性、三角函数、幂、指数、对数、有理分式等）以加法、乘法、除法、一元包装等方式附着到父代。
     - **规则删除**：修剪或展开AST子结构。
     - **LLM引导的变异**：接收父代公式、标注AST、拟合参数、规则生成候选摘要、领域知识、历史最优表达式，生成非模板、上下文相关的ADD（保留父代附加新内容）或SUBST（替换错误子结构）变异。
   - **Optimizer（结构感知系数优化器）**：利用表达式结构进行多阶段拟合：
     - **变量投影**：分离线性/非线性参数，线性参数用OLS解，只搜索非线性子空间。
     - **结构预搜索**：对复合幂、偏移中心等难优化参数预先枚举候选值（如整数/有理数指数）。
     - **全局搜索**：trust-region最小二乘、CMA-ES、差分进化。
     - **局部精化**：L-BFGS-B。
     - **参数快照与重拟合**：对指数类参数快照到最近整数/有理数，仅当重拟合后目标不恶化才接受。
     - **符号简化**：合并冗余参数，消除零项、单位简化等。
   - 每个生成的候选先经过归一化和结构去重（基于参数名不变的结构指纹），再进入优化器，最后经过退化检查（大系数过拟合、简化等）决定是否加入进化树。

#### 算法流程（文字描述）
1. 调用LLM Generator获取领域知识K并生成种子集S₀。
2. 对每个种子归一化、优化、评估后加入表达式数据库E。
3. 每轮迭代：
   - 从E提取进化树摘要Z。
   - LLM Selector基于Z选择P个父代。
   - 对每个父代，应用规则删除、规则添加、LLM变异，生成候选集C。
   - 对每个候选归一化、去重、优化、退化检查，将有效节点加入E。
   - 更新选择历史H；当成熟节点（训练NMSE低于阈值）数达到目标时提前停止。
4. 返按照训练NMSE排序的top表达式。

## 6. 主要结论与发现
1. **性能大幅提升**：在LLM-SRBench 129任务上，FunctionEvolve（Claude Opus 4.6）达到SA@50=107（82.9%），是同等骨干基线（LLM-SR和OpenEvolve均为24）的4.5倍；SA@1=72，是此前最强PiT-PO（20）的3.6倍。数值精度接近真值（中位测试NMSE 1.38e-13）。
2. **结构可见搜索是核心**：移除所有AST结构信息（w/o AST Structure）后SA@50从107降至60，降幅显著，尤其物理子集受严重影响。
3. **LLM局部精化和结构感知优化均关键**：移除LLM Mutator或结构感知优化器分别导致SA@50降到46和53；组合移除更是降到34。
4. **规则变异已具较强基础**：纯规则变异（No LLM）已达到SA@50=29，接近PiT-PO的20 top-1结果，表明结构感知拟合和规则搜索本身是强基线。
5. **噪声敏感**：1%噪声下SA@50降至54，5%降至40，符合预期。
6. **基准局限性**：材料科学子集的温度T与应变ϵ存在几乎完全线性相关（残差最大4.43e-5），且Arrhenius因子变化极小，导致结构不可辨识。其他领域部分任务也存在高变量相关性。

## 7. 优点
- **创新性**：首次将显式表达式树（AST）贯穿LLM驱动的符号回归全流程（选择、变异、系数优化），解决了此前系统的结构盲区。
- **鲁棒性**：结合确定性规则变异与LLM引导变异，既保证局部编辑的系统性，又利用LLM的上下文灵活性。
- **系数优化深度利用结构**：变量投影分离线性/非线性参数、结构预搜索（幂指数、偏移中心）、快照重拟合等策略，使正确骨架更易被保留。
- **通用性**：只使用初等函数族，无领域特定规则，降低归纳偏置，可泛化到不同领域。
- **效率**：LLM调用次数远少于基线，价格加权成本相当或更低。
- **基准审计**：暴露了LLM-SRBench的材料科学子集的可辨识性问题，有助于社区改进基准设计。
- **实验全面**：包含多种骨干、消融、噪声、复杂度选择、优化器基准、AI-Feynman验证，且对LLM评判质量进行了人工审计。

## 8. 不足与局限
- **噪声鲁棒性不足**：在有噪声的真实场景下，符号恢复准确率急剧下降，缺乏不确定性感知评分机制。
- **最终排名问题**：SA@50与SA@1差距较大（107 vs 72），说明正确形式常被数值更强但结构不简洁的表达式压倒，复杂度感知选择仍无法完美识别。
- **变量集规模限制**：论文未测试更大候选变量集（如>10维）或动态系统/无闭式解方程。
- **基准依赖**：主要结果依赖LLM-SRBench合成基准，该基准存在可辨识性问题（材料科学）和任务间的变量相关性，可能影响部分结论的泛化性。
- **计算资源说明不完整**：未报告端到端训练时间，仅给出每任务迭代数和超时。本地CPU部署细节（如多任务并行度）未量化。
- **调优风险**：虽然声称无领域特定规则，但规则添加库的算子组合、优化器超参（如预搜索候选值集）可能存在隐含的手动设计，影响公平性。
- **模型版本敏感性**：结果高度依赖于Claude Opus 4.6等前沿模型，较弱骨干（如Llama-3.1-8B）提升有限，且JSON格式遵循能力弱时需回退规则基线。
