## 摘要
我们提出了PDE-Agents，一个多智能体生态系统，通过自然语言交互自动化偏微分方程（PDE）/有限元方法（FEM）仿真的完整生命周期。三个专业的大型语言模型（LLM）智能体（仿真、分析、数据库）通过LangGraph监督器进行编排，并在双NVIDIA RTX PRO 6000 GPU上使用本地开源LLM堆栈（Qwen3-Coder-Next, Llama 4 Scout）。该架构与模型无关，并在两代LLM上进行了验证。一个GraphRAG知识库（Neo4j，768维向量嵌入）编码了精选的材料属性、已知的失效模式和先前运行谱系。我们报告了七项贡献：(i) 一项验证与确认（V&V）研究，确认了热方程求解器的二阶空间收敛性（O(h^2)）；(ii) 在冻结知识图谱（KG On, KG Off, KG Smart）上对50个任务进行的三路消融实验，其中KG Smart达到100%成功率及最高输出质量（物理0.933对比KG Off的0.853；MPF 0.926对比0.796）；(iii) 一项使用三种仅KG已知的虚构材料的新材料实验，其中KG Smart实现了近乎完美的材料属性保真度（MPF = 1.00），而无KG基线仅为0.34；(iv) 一项故障分析，将KG On的三次失败归因于预算耗尽和超时，确立热启动注入为主要可靠性因素；(v) 一个为每个任务选择最优检索模式的适应性框架；(vi) 来自1369次运行的生产指标（97.8%成功率，57.6%首次尝试成功）；(vii) 一项100任务的知识图谱增长实验，显示出难度依赖的增益，其中困难任务的MPF提高了8.8%，而简单/新颖任务保持在最高水平。所有代码、模型和评估工件均已开源。我们的发现表明，是整合模式而非知识内容决定了GraphRAG增强是否有助于或阻碍LLM智能体。

## Abstract
We present PDE-Agents, a multi-agent ecosystem that automates the full lifecycle of partial differential equation (PDE) / finite element method (FEM) simulations through natural-language interaction. Three specialist large language model (LLM) agents (Simulation, Analytics, Database) are orchestrated via a LangGraph supervisor, with a local open-source LLM stack (Qwen3-Coder-Next, Llama 4 Scout) on dual NVIDIA RTX PRO 6000 GPUs. The architecture is model-agnostic, validated across two LLM generations. A GraphRAG knowledge base (Neo4j, 768-d vector embeddings) encodes curated material properties, known failure patterns, and prior run lineage. We report seven contributions: (i) a verification and validation (V&V) study confirming second-order spatial convergence (O(h^2)) on the heat-equation solver; (ii) a three-way ablation over 50 tasks with a frozen KG (KG On, KG Off, KG Smart), where KG Smart reaches 100% success and the highest output quality (physics 0.933 vs. 0.853 for KG Off; MPF 0.926 vs. 0.796); (iii) a novel-material experiment with three fictional materials known only to the KG, where KG Smart attains near-perfect material property fidelity (MPF = 1.00) versus 0.34 for the KG-free baseline; (iv) a failure analysis tracing KG On's three failures to budget exhaustion and timeout, establishing warm-start injection as the dominant reliability factor; (v) an adaptive framework selecting the optimal retrieval mode per task; (vi) production metrics from 1,369 runs (97.8% success, 57.6% first-try); and (vii) a 100-task KG growth experiment showing a difficulty-dependent gain, with hard-task MPF improving 8.8% while easy/novel tasks stay at ceiling. All code, models, and evaluation artifacts are released openly. Our findings show that integration pattern, not knowledge content, determines whether GraphRAG augmentation helps or hinders LLM agents.

## 2. 论文提出的方法论：核心思想、关键技术细节
### 2.1 核心思想
- **多智能体编排**：通过LangGraph监督器协调三个专业LLM智能体（仿真、分析、数据库），实现任务解耦和闭环调试。
- **知识图谱增强**：集成Neo4j图数据库（GraphRAG），利用HNSW向量索引支持语义检索，为智能体提供材料属性、已知故障模式、历史运行谱系。
- **KG Smart模式**：采用“热启动注入 + 惰性条件检索”策略，避免强制KG查询导致的预算耗尽问题，在可靠性和输出保真度间取得平衡。

### 2.2 关键技术细节
- **Agent栈**：
  - Orchestrator：LangGraph StateGraph，接收用户自然语言请求，路由到合适Agent并合成最终响应。
  - Simulation Agent：核心推理Agent，采用ReAct循环（最多25步），拥有9个工具（`check_config_warnings`, `query_knowledge_graph`, `validate_config`, `run_simulation`, `modify_config`, `debug_simulation`, `list_recent_runs`, `get_run_status`, `run_parametric_sweep`）。
  - Analytics Agent：统计比较、敏感性分析、文本摘要。
  - Database Agent：PostgreSQL查询历史、目录化结果。
- **FEM求解器**：基于DOLFINx 0.10.0.post2（FEniCSx），求解稳态和瞬态热方程，支持P1（线性Lagrange）单元，Backward Euler时间积分，Dirichlet/Neumann/Robin边界条件。几何库包括2D矩形、L形、圆、环形、T形、阶梯缺口等以及3D模型，使用Gmsh生成网格。
- **知识图谱架构**：
  - Neo4j 5 Community Edition。
  - 节点类型：Material（材料）、Run（运行记录，带768维嵌入）、KnownIssue（已知故障模式）、ReferenceChunk（文档片段）等。
  - 边：USES_MATERIAL, SIMILAR_TO（余弦相似度≥0.85），CROSS_REFS等。
  - 嵌入模型：`nomic-embed-text`（768维），HNSW索引。
  - 文档智能管线：Docling提取PDF/HTML，分块并自动关联。
- **KG Smart模式**（Algorithm 1）：
  - 若任务描述包含显式参数 → KG Off。
  - 若材料已知 → KG Smart（惰性检索）。
  - 若材料未知 → KG Smart（强制热启动）。
  - 热启动：任务描述经`nomic-embed-text`嵌入，在Neo4j HNSW索引中检索Top-3相似历史成功运行，将其配置注入Agent提示。
- **工具调用兼容性**：针对不同LLM家族（如Qwen系列工具调用格式差异）实现四阶段标准化解析（JSON解析、Markdown fence提取、括号修补、JSON双编码递归解除）。

## 6. 论文的主要结论与发现
1. **KG Smart模式达到最佳平衡**：在50任务消融中，KG Smart实现100%成功率和最高输出质量（Physics Score 0.933, MPF 0.926），同时可靠性匹敌KG Off，保真度优于KG On。
2. **故障分析揭示热启动是主导因素**：KG On的3个系统性失败全部源于强制KG查询导致预算耗尽/超时，而KG Smart的热启动注入消除了此瓶颈。惰性检索仅用于故障恢复，次要贡献。
3. **新材料场景下KG不可或缺**：NG Smart在新材料上达到MPF=1.00，而KG Off仅0.34。KG Off因LLM幻觉产生的错误属性导致输出温度误差最高达949 K（47%偏离）。
4. **KG价值具有难度依赖性**：KG增长实验显示，易/中等任务在pass1已接近天花板（MPF≥0.96），而难任务在pass2 MPF提高8.8%（0.783→0.870），Physics Score提高5.8%。
5. **生产级可靠性**：1369次运行整体成功率97.8%，首次尝试成功率57.6%，通过多轮自纠正提升。
6. **模型无关性**：跨两种LLM验证，KG Smart均达到完美MPF，确认依赖的是检索而非模型记忆。
7. **设计原则**：
   - 永远不要将KG访问放在Agent关键路径上（强制查询导致脆弱耦合）。
   - 通过语义相似性前置上下文（热启动），而非要求Agent花费迭代去查询。
   - 让Agent自主决定何时需要更多知识（惰性条件检索）。

## 8. 不足与局限
- **PDE范围有限**：当前仅支持标量热方程，未扩展到矢量力学（如线弹性、Navier-Stokes），需要更丰富工具模式。
- **V&V覆盖面不足**：未包括高阶元素、含奇异性域、时间收敛研究；三维情况仅展示个别示例。
- **缺乏并行性**：Agent顺序执行，未实现任务级并行（如参数扫描并行）。
- **知识图谱建设尚浅**：当前仅支持SIMILAR_TO边，未来可挖掘IMPROVED_OVER关系（如网格细化效益）、故障诊断边，以及引用可追溯性。
- **系统依赖人工工程**：Agent提示词、工具定义等仍需要大量人工调整，自动化程度可进一步提升。
